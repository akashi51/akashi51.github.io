# 需求文档

## 1. 应用概述

- **应用名称**：个人主页
- **应用描述**：一款现代简约风格的个人主页，适用于 GitHub 个人展示页面，包含导航、个人介绍、技能标签、项目展示及联系方式等核心模块，整体风格简洁、专业。

---

## 2. 用户与使用场景

- **目标用户**：开发者个人，用于在 GitHub 上展示个人信息与项目成果
- **核心使用场景**：访客通过 GitHub 链接访问该主页，快速了解作者背景、技能方向与代表项目，并获取联系方式

---

## 3. 配色方案

采用现代简约风格，以深色背景搭配高亮强调色，营造专业、清晰的视觉层次：

| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | #0F1117 | 深夜黑，主背景色 |
| 卡片/区块背景 | #1A1D27 | 深灰蓝，内容区块底色 |
| 主强调色 | #6C63FF | 紫罗兰，用于按钮、链接、标签高亮 |
| 次强调色 | #00D4AA | 青绿色，用于技能标签、悬停效果 |
| 主文字色 | #E8EAF0 | 近白色，正文主色 |
| 次文字色 | #8B8FA8 | 灰蓝色，副标题、描述文字 |
| 分割线/边框 | #2A2D3E | 深灰，区块边框与分割线 |

---

## 4. 页面结构与功能说明

### 4.1 整体页面结构

```
个人主页（单页滚动）
├── 顶部导航栏（固定吸顶）
├── 个人介绍区（Hero Section）
├── 技能标签区
├── 项目展示区
└── 底部联系方式区
```

### 4.2 顶部导航栏

- 固定吸顶，滚动时保持可见
- 左侧：个人名称或 Logo 文字（占位：「Your Name」）
- 右侧：锚点导航链接，依次为：关于我 / 技能 / 项目 / 联系
- 点击导航链接，页面平滑滚动至对应区块
- 背景：半透明深色背景，带轻微模糊效果（backdrop blur）
- 导航链接悬停时显示主强调色下划线动效

### 4.3 个人介绍区（Hero Section）

- 布局：左侧文字信息 + 右侧头像占位符（居中对齐，响应式两栏）
- 头像占位符：圆形，尺寸 160×160px，边框使用主强调色渐变描边，内部显示默认人物轮廓图标
- 文字内容（均为占位文案）：
  - 大标题：「Hi, I'm Your Name 👋」
  - 副标题：「Full Stack Developer / Open Source Enthusiast」
  - 简介段落：2-3 行个人描述占位文字
- 行动按钮：「查看我的项目」（主强调色填充）+ 「联系我」（描边按钮）
- 区块背景：页面主背景色，顶部留有充足留白

### 4.4 技能标签区

- 区块标题：「技能 & 工具」
- 展示形式：标签云（Flex 换行布局），每个标签为圆角胶囊形
- 标签分两类样式：
  - 主技能标签：主强调色背景，白色文字（示例：JavaScript、React、Node.js）
  - 工具标签：次强调色描边，次强调色文字（示例：Git、Docker、VS Code）
- 占位标签数量：10-12 个
- 标签悬停时有轻微放大动效（scale 1.05）

### 4.5 项目展示区

- 区块标题：「我的项目」
- 布局：三列卡片网格（移动端降为单列）
- 每张项目卡片包含：
  - 项目名称（加粗标题）
  - 项目描述（2 行文字，超出省略）
  - 技术标签（1-3 个小标签，次强调色）
  - 底部操作行：「GitHub」链接图标 + 「预览」链接图标
- 卡片背景：卡片/区块背景色，圆角 12px，悬停时边框变为主强调色并有轻微上浮阴影
- 占位项目数量：3 张卡片

### 4.6 底部联系方式区

- 区块标题：「联系我」
- 副标题：「欢迎交流合作，随时联系」
- 联系方式列表（图标 + 文字，横向排列）：
  - GitHub：链接占位
  - 邮箱：邮箱地址占位
  - LinkedIn：链接占位（可选展示）
- 每项联系方式悬停时文字变为主强调色
- 底部版权信息：「© 2026 Your Name. All rights reserved.」
- 区块背景：卡片/区块背景色，与主背景形成轻微区分

---

## 5. 业务规则与逻辑

- 所有导航锚点链接对应页面内各区块的 id，点击后平滑滚动（scroll-behavior: smooth）
- 「查看我的项目」按钮点击后平滑滚动至项目展示区
- 「联系我」按钮点击后平滑滚动至底部联系方式区
- 项目卡片中的 GitHub 与预览链接均在新标签页打开（target=_blank）
- 页面整体为单页滚动结构，无路由跳转

---

## 6. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 头像图片未设置 | 显示默认圆形占位图标，不破坏布局 |
| 项目描述文字过长 | 超出两行后以省略号截断 |
| 联系链接为空 | 对应联系项不渲染，不显示空白占位 |
| 移动端窄屏 | 导航栏折叠为汉堡菜单，卡片网格降为单列 |

---

## 7. 验收标准

- 页面在桌面端（1280px 以上）与移动端（375px）均可正常展示，布局无错位
- 顶部导航栏固定吸顶，滚动时不遮挡内容
- 所有锚点导航点击后可平滑滚动至对应区块
- 配色方案与设计规范一致，无色值偏差
- 项目卡片悬停效果、技能标签悬停效果正常触发
- 所有占位文案清晰可辨，结构完整
